## 🌟 نبذة عن النظام (Executive Overview)

**RestoHub** هو نظام ERP ونقطة بيع (POS) سحابي متطور، مصمم خصيصاً ليعمل بنموذج **SaaS متعدد المستأجرين (Multi-Tenant)**. يتيح النظام لإدارة المنصة تأجير الخدمة لمئات المطاعم والمقاهي، بحيث يمتلك كل مطعم فروعاً وصالات ومخزوناً وطاقم عمل معزولاً ومحمياً بالكامل عن باقي المشتركين.

تمت إعادة هندسة هذا المشروع معمارياً بمعايير **Clean Enterprise Architecture** والانتقال به من مرحلة التقييم المبدئي إلى بنية **Cloud-Native Backend-as-a-Service (BaaS)** بالاعتماد المباشر والأصيل على **Supabase** و **Vercel**، متخلصين بالكامل من أي خوادم وسيطة أو تعقيدات صيانة (Zero DevOps).

## 🏗️ التصميم المعماري النظيف (Architectural Highlights)

```
+-------------------------------------------------------------+
|              Vercel Cloud Edge / Client Browser             |
|       [ React 19 + Vite + Tailwind CSS + Lucide Icons ]     |
+-------------------------------------------------------------+
                               |
                               |  Explicit Type-Safe DTOs (toTenant, toOrder...)
                               v
+-------------------------------------------------------------+
|              Supabase Native Cloud Infrastructure           |
|  +-----------------------+     +-------------------------+  |
|  |     Supabase Auth     |     |  PostgreSQL Engine RLS  |  |
|  |  (JWT & Secure Sess.) |     |  (Tenant Data Isolation)|  |
|  +-----------------------+     +-------------------------+  |
+-------------------------------------------------------------+
```

1. **الاستغناء عن الخادم الوسيط (Zero-Backend Clean Architecture):**
   تم حذف خادم Express المكرر ليصبح التطبيق واجهة React نقية وفائقة السرعة تتحدث مباشرة مع قاعدة بيانات Supabase عبر مكتبتها الرسمية `@supabase/supabase-js`.
2. **محولات البيانات الصريحة (Explicit Type-Safe DTO Mappers):**
   تم التخلص تماماً من أي حلول ترقيعية أو انعكاسية عشوائية (Runtime Reflection)، وبناء طبقة بيانات برمجية صريحة تحوّل سجلات قاعدة البيانات (`snake_case` مثل `tenant_id`, `name_ar`) إلى نماذج واجهة المستخدم (`camelCase` مثل `tenantId`, `nameAr`) عبر محوّلات مخصصة لكل كيان (`toTenant/fromTenant`, `toOrder/fromOrder`).
3. **الأمان السيادي وعزل المستأجرين (Row Level Security - RLS):**
   تعتمد الحماية على محرك قاعدة البيانات نفسه؛ حيث تضمن سياسات الـ RLS المكتوبة في `schema.sql` عدم قدرة أي موظف أو كاشير على الوصول إلى بيانات خارج النطاق الجغرافي والإداري لمطعمه (`tenant_id`).
4. **الوضع الاحتياطي الذكي (Smart Offline/Demo Fallback):**
   في حال تشغيل المشروع محلياً دون إدخال مفاتيح Supabase، يعمل النظام بسلاسة على وضع التجربة المحلي محتفظاً بالبيانات في التخزين المحلي دون أي انهيار.

## 📦 الوحدات الوظيفية الرئيسية (Core Modules)

| الوحدة | الوصف والوظائف الرئيسية |
| :--- | :--- |
| **🍔 شاشة الكاشير ونقطة البيع (POS)** | واجهة سريعة الاستجابة تدعم أنواع الطلبات المختلفة (محلي داخل الصالة، سفري، وتوصيل خارجي) مع ربط مباشر بالفواتير والضرائب. |
| **📺 شاشة المطبخ الذكية (KDS)** | استقبال الطلبات في الوقت الحقيقي للمطبخ مع تتبع حالات التحضير والتجهيز والتسليم. |
| **🪑 خريطة الصالات والطاولات (Tables & QR)** | تقسيم المطعم لفروع وصالات، وتتبع حالة الطاولات (متاحة، مشغولة، محجوزة) مع توليد رموز QR للطلب الذاتي من المنيو الرقمي. |
| **📦 المخزون وحساب التكاليف (Inventory & Recipes)** | ربط الأطباق بالمكونات الخام (Ingredients) وخصم المخزون تلقائياً عند البيع مع تنبيهات وصول المخزون للحد الأدنى. |
| **👥 شؤون الموظفين والرواتب (HR & Payroll)** | إدارة صلاحيات الطاقم (مدير، كاشير، طباخ، نادل)، وسجل الحضور والتقييم، وحساب كشوف الرواتب الشهرية والخصومات والمكافآت. |
| **💼 المحاسبة وسجل التدقيق (Accounting & Audit)** | تسجيل الإيرادات والمصروفات المالية تلقائياً، وتوثيق أي حركة حساسة في النظام (مسح طلب، تعديل سعر) ضمن سجل تدقيق غير قابل للمسح. |
| **👑 لوحة تحكم السوبر أدمن (System Admin)** | واجهة سيادية لإدارة المنصة، مراقبة الاشتراكات (Pro, Enterprise)، وإنشاء المستأجرين والفروع الجديدة. |

## 🚀 دليل التشغيل المحلي والنشر السحابي (Quick Start & Deployment)

### 1️⃣ التشغيل والتطوير المحلي (Local Development)

قم باستنساخ المستودع والدخول للمجلد الرئيسي ثم تثبيت الحزم وتشغيل خادم التطوير:

```bash
git clone https://github.com/oday023/restohub-db-updates.git
cd restohub-db-updates
npm install
npm run dev
```

افتح المتصفح على الرابط `http://localhost:3000` لتجربة النظام فوراً.

---

### 2️⃣ إعداد قاعدة البيانات على Supabase

1. قم بإنشاء مشروع جديد في منصة [Supabase](https://supabase.com).
2. افتح تبويب **SQL Editor** في لوحة تحكم Supabase.
3. انسخ محتوى الملف **`schema.sql`** الموجود في جذر هذا المشروع بالكامل، والصقه في المحرر ثم اضغط **Run**.
4. سينشئ هذا الأمر جميع الجداول، الفهارس، علاقات الربط، ودوال التشفير (`pgcrypto`) وحسابات التجربة الأولية.

---

### 3️⃣ النشر بضغطة زر على Vercel

1. ارفع المشروع إلى حسابك في GitHub.
2. ادخل إلى لوحة تحكم [Vercel](https://vercel.com) واضغط على **Add New Project** ثم استورد المستودع.
3. في شاشة إعدادات النشر (Environment Variables)، أضف المتغيرين التاليين من إعدادات مشروعك في Supabase (تبويب Settings -> API):
   * **`VITE_SUPABASE_URL`**: رابط مشروعك في Supabase.
   * **`VITE_SUPABASE_ANON_KEY`**: المفتاح العام (Public Anon Key).
4. اضغط على **Deploy**. سيقوم Vercel بإنشاء التجميع النهائي (`vite build`) ونشر منصة SaaS الرسمية الخاصة بك في ثوانٍ معدودة.
