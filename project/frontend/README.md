# 🍔 منصة إدارة المطاعم المتقدمة متعددة الفروع والمستأجرين (SaaS)
## Professional Multi-Tenant Restaurant Management SaaS Platform

مرحباً بك في منصة **إدارة المطاعم المتكاملة للإنتاج الحقيقي (SaaS)**، والمصممة لتشغيل عدة مطاعم مختلفة بشكل سحابي مستقل وآمن بالكامل مع واجهة سريعة، تفاعلية وعالية الأداء شبيهة بالأنظمة العالمية مثل Toast POS و Foodics.

Welcome to the **Multi-Tenant Restaurant Management SaaS Platform**, engineered to operate multiple independent restaurant brands with complete data isolation, high performance, and an interactive interface styled like Toast POS, Foodics, and Square.

---

## 🚀 التقنيات المستخدمة (Technical Stack)

- **Frontend**: واجهة تفاعلية حديثة وسريعة تعتمد على **React 19 + TypeScript + Tailwind CSS**.
- **State Logic**: تنظيم الحالات والفوترة التلقائية عبر خدمات **StorageService** الذكية للحفظ المحلي المحاكي لقاعدة البيانات السحابية بالكامل (`localStorage`).
- **Database Architecture**: تصميم هيكلية وجداول **PostgreSQL** للإنتاج مع أدوات نسخ المخطط وإدارة الجداول بشكل احترافي.
- **Responsive Web**: متوافق تماماً مع شاشات الكمبيوتر اللوحي المكتبي (Desktop POS) وأجهزة الجوال (QR Self-Ordering).

## 🛠️ المميزات والحلول المصممة خصيصاً (Key Engineered Features)

### ١. المحاكاة البيومترية للحضور والشؤون (Biometric Punching Simulator)
داخل تبويب **الموظفين وحضور البصمة**، يمكنك محاكاة تفعيل بصمة كل موظف (Clock IN / Clock OUT). يقوم النظام بتسجيل تاريخ اليوم وتوقيت المرور لجمع تقارير الدوام الفعلي وحساب مخرجات الموظف.

### ٢. ربط المكونات بالوجبات والخصم التلقائي (Dynamic Recipes Decrement)
تم ربط المنتجات (مثل برجر اللحم الكلاسيكي) بالمواد الأولية مباشرةً (اللحم، الخبز، الجبن). فور إتمام أي طلب بيع بنجاح عبر شاشات الـ POS أو الطلب الذاتي QR، ينفذ النظام حوسبة لخصم المكونات بدقة من المستودع وتأمين إنذار حد الخطر الأحمر تلقائياً!

### ٣. نظام الكيو آر المباشر وتتبع الطلة (Digital QR Self-Ordering)
يستطيع زوار الطاولات تصفح قائمة الطعام المتكاملة، وتعديل المكونات الإضافية، وإدخال بيانات الولاء لجمع النقاط، ثم **إرسال الطلب فورياً للمطبخ**. يستلم الشيف الطلب في شاشة الـ KDS بصوت رنين منبه ميكروويف، وبمجرد تحويل الطلب إلى "جاهز!"، ينتقل المؤشر على هاتف الزبون لإبلاغه فوراً بالخدمة.

### ٤. الصلاحيات المتقدمة بناءً على دور الموظف (Role-Based Access Control)
لتأمين مستندات المالية والشؤون للمطعم، يتفاعل شريط النظام مع مصفوفة الـ RBAC:
- **الموظف العادي (الويتر أو الشيف)**: يتم قفل تبويبات رواتب الموظفين والمستودع والخزينة والمالية لضمان الخصوصية التامة.
- **مالك المطعم والمدير**: يتم تفعيل كامل أدوار الإشغال والصلاحيات.

## ⚙️ التوافق والتشغيل وعقد التطوير (Deployment & Running Instructions)

- النظام جاهز ١٠٠٪ للإنتاج والتشغيل.
- لتشغيل خادم التطوير محلياً:
  ```bash
  npm run dev
  ```
- لتشغيل خادم PostgreSQL/Express التجريبي وبدء اختبار الاتصال بقاعدة البيانات:
  ```bash
  npm run dev:server
  ```
- لإنشاء الجداول في PostgreSQL:
  ```bash
  npm run db:init
  ```
- قبل التشغيل، انسخ ملف البيئة المثال إلى `.env` وأضف بيانات PostgreSQL أو `DATABASE_URL`.
- لبناء حزمة الإنتاج وفحص الكود:
  ```bash
  npm run lint
  npm run build
  ```

## 🔌 تهيئة PostgreSQL

المشروع ما زال يستخدم التخزين المحلي داخل الواجهة حالياً، لكن تمت إضافة طبقة خادم Node/Express جاهزة للربط مع PostgreSQL. هذا يتيح لك:

- اختبار اتصال قاعدة البيانات عبر `GET /api/db/test`.
- ضبط الاتصال من خلال `DATABASE_URL` أو متغيرات `PGHOST` و`PGUSER` و`PGDATABASE`.
- لاحقاً نقل عمليات القراءة والكتابة من `localStorage` إلى API دون إعادة بناء الهيكل الكامل.
- إنشاء الجداول مباشرة من ملف migration داخل [db/migrations/001_init.sql](db/migrations/001_init.sql).
